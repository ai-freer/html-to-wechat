改写示例：从技术博客到 30 秒短视频口播

同样一段内容，输入是结构化技术写作，输出是适合朗读 / TTS 的口语片段。

输入 · HTML 文章片段

## L6 会话生存层

**解决的问题**：长对话会超过
模型上下文窗口。只靠简单截断
或一次性 compaction，会让早期
约定、任务状态和关键决策从当
前对话里消失。

**OpenClaw 中的实现**：
- Lossless Context Engine：
  实时摄入消息，并用 DAG/摘要
  /分层压缩组装下一轮模型上下文
- Session Memory hook：实时
  保存会话 transcript

输出 · 口播稿 .md

咱们聊聊 L6——会话生存层。

很多人以为，只要模型窗口够大，
就不用管上下文了。

但你跑一个长任务三天后，最早
那几句关键约定，已经被截断
丢掉了。

OpenClaw 是怎么解决的？

它有一个东西叫 Lossless
Context Engine。实时摄入每
一条消息，用 DAG、摘要、
分层压缩，组装下一轮的上下文。

简单说：长对话不丢东西。

工作流

Skill 在你的 agent 里跑 LLM 改写，浏览器只做最后的预览 / 复制 / 跳转 V2C 工具。

1

在 agent 里装 Skill

给你的 Claude / Codex 装上 html-to-wechat Skill 包，包含改写口播稿的提示链。

2

跑一条命令

把 HTML / Markdown 文件丢给 agent，例如 /h2w-script article.html。agent 用 LLM 改写并估算朗读时长。

3

本页面预览 + 复制 / 下载

agent 跑完后浏览器打开本页面，看到渲染后的 markdown + 字数 / 时长估算 + 常用 V2C 工具快捷外链。
