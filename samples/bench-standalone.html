横向评测 Bench·Report·2026

北京车展重点车型 × 自有 AI 汽车能力 Benchmark
Auto China 2026 — Cockpit-LLM & Agent Capability Benchmark

行业正在从「车载问答」走向「车载智能体」。 但多数车型仍停留在大模型问答、多模态理解、车控导航或生态接入阶段；真正能形成差异化的，是 复杂任务规划、跨 App 执行、生态闭环 Agent、AI 画布、长期记忆和情绪化体验 的组合闭环。自有内容的核心机会不是「我们也有大模型」，而是—— AI 替用户动手完成现实任务。

POSITIONING

本报告虽对应「精简版」，但不是 Full Report 的缩略摘要。Full Report 关注北京车展汽车 AI 的完整行业复盘；Bench Report 关注更窄但更深入的问题—— 其他车型上的 AI 大模型能力到底做到哪一步？它们与自有内容相比，在哪些能力上重叠、在哪些能力上存在差距、哪些差异可以转化成对外传播和产品定义？

因此「精简」只体现在范围收窄：不展开完整智驾产业链、不复述全部车展事实、不堆全部车型细表。但在大模型评测范围内，评测框架、对比对象、能力等级、自有能力差距、证据等级与待核验项必须完整。

01

八条结论Key Takeaways

先给出评测结果，再展开评分方法和证据。每条结论都对应后文章节。

「真正能形成差异化的，是复杂任务规划、跨 App 执行、生态闭环 Agent、AI 画布、长期记忆和情绪化体验的组合闭环。」

— 总判断（§00）
  1. 01

    大模型上车已经成为行业共识，但能力深度差异大。

    从传统语音到大模型问答、从多模态感知到 Agentic 执行，各品牌实际落点不同。C0–C6 阶梯把它们拉到同一根尺子上比较。

    对应 §03 / §06
  2. 02

    Agentic AI 是本届车展最重要的座舱关键词。

    豆包座舱助手（自有）、华为、大众、阿里、腾讯、讯飞等都在不同程度上强调智能体、任务规划和生态调用。叙事高度趋同，差异在执行深度。

    对应 §06.2 §06.4 §06.5 §06.6 §06.7 §06.8
  3. 03

    生态闭环 Agent · 跨 App 类人操作仍是明显稀缺能力。

    多数公开案例偏问答、导航、车控、内容调用和生态入口；能像人一样跨 App 跑完一整段任务闭环的强公开案例较少。自有方案走协议级生态共建路线，节奏取决于合作进度。

    对应 §07 生态闭环 Agent / §08 Gap Matrix 顶行
  4. 04

    外资品牌的中国本地化 AI 值得重点对比。

    BMW、大众、奔驰、奥迪等开始接入中国大模型、地图生态和本土智驾供应链——中国智能座舱体验正在反向定义豪华品牌本地方案。

    对应 §06.1 §06.2 §06.3
  5. 05

    华为是系统完整度最强的参照对象。

    覆盖鸿蒙座舱、小艺、盘古、AMS、ADS、激光雷达和车型矩阵。但需要把座舱模型和智驾模型分开评估，避免混成一个能力。

    对应 §06.4
  6. 06

    讯飞、阿里、腾讯、豆包座舱助手（自有）代表四种不同座舱 Agent 路线。

    讯飞偏端侧多模态与语音工程化；阿里偏端云协同和生活服务生态；腾讯偏微信/小程序/支付生态组织；豆包座舱助手偏 Agentic AI、内容生态与人格化体验，自有方案以此对外（详见 §06.5 / §07）。

    对应 §06.5–§06.8
  7. 07

    自有方案的强表达应是「任务闭环 + 情绪体验」。

    只讲能聊天不够，只讲情绪陪伴偏软。最强组合是 Planner + 生态闭环 Agent + AI 地图 Agent 证明能办事，再用活人感、记忆和座舱联动证明懂人。

    对应 §07 三层结构 / §09
  8. 08

    评测必须保留证据等级。

    车展期存在大量品牌级合作、供应商口径、生态搭载、概念演示和媒体转述。所有竞品能力都应区分 Current Level 和 Potential Level。

    对应 §02 §10
02

评测框架Evaluation Framework

先定义"怎么比"，否则横向对比会变成主观印象。

§2.1五条评测原则

  1. 只比较可感知能力，不只比较模型名。

    用户不会因为「接入某大模型」产生强感知，用户会因为 AI 真正帮他完成任务、减少操作、理解情绪、记住习惯而感知差异。

  2. 区分能力声明和已交付能力。

    概念车、预售车、供应商 demo、OTA 承诺和量产配置不能混写。

  3. 区分座舱大模型和智驾模型。

    VLA、World Model、End-to-End 是重要背景，但本报告主评测对象是座舱大模型和车载 Agent 能力。

  4. 区分生态接入和 AI 执行。

    地图、音乐、支付、小程序、内容 App 上车，不自动等于 AI 能主动调度这些工具完成任务。

  5. 所有判断带证据等级。

    A/B/C/D 标注不是形式，而是避免把营销话术写成事实。

§2.2纳入对象标准

IN 主评测对象，至少满足一项

  • 明确提到大模型、LLM、VLM、多模态模型。
  • 明确提到 AI 智能体、Agentic AI、任务规划。
  • 明确提到座舱多模态感知和大模型交互。
  • 明确有本地模型合作方或座舱 AI 供应商。
  • 与自有能力存在可比性，如地图 Agent、人格化助手、主动提醒、跨生态执行、长期记忆。

OUT 不纳入主评测，可在附录提及

  • 纯智驾算法发布。
  • 纯芯片或传感器发布。
  • 纯车联网生态或 App 预装。
  • 传统语音助手升级但未见大模型证据。
SCOPE

结构化数据建议增加 comparison_scope 字段：primary（进入主评分矩阵）/ observer（进入观察表，不打 C0–C6）/ self_reference（自有能力基准/对照锚点，不作为竞品）。

03

C0–C6 能力阶梯Capability Ladder

用一套清晰等级判断竞品做到哪一步。本阶梯与 Full Report 的 C0–C6 一一对应。

  1. C6

    主动智能体

    类人操作跨 App 任务闭环 + 记忆 / 个性化 / 主动预测——可像人一样操作第三方 App、处理异常并主动给出下一步（生态协议级集成或视觉捕捉皆可达成）。

    用户感知AI 主动替你跑完整段

    示例订餐、买券、叫跑腿、主动提醒

  2. C5

    跨生态闭环

    可连接地图、本地生活、内容、支付等生态完成任务。

    用户感知车能替你跑流程

    示例搜店、订座、导航

  3. C4

    多域多步骤

    可跨多个领域拆解复杂任务、串联多步骤工具调用并反馈。

    用户感知车能安排一整件事

    示例规划接人、吃饭、停车

  4. C3

    单域 Agentic 任务执行

    具备大模型的代理性，能在车控、导航、内容等单一领域内智能化理解并执行一定复杂度的任务（区别于 C0 规则式语音控制）。

    用户感知车能聪明地做一件事

    示例智能导航、按情境播放、复杂车控

  5. C2

    多模态理解

    理解语音、视觉、座舱状态或乘员身份。

    用户感知车能看懂场景

    示例识别乘员、理解情绪

  6. C1

    大模型问答

    可自然语言问答，但执行弱。

    用户感知车能聊天

    示例问百科、问车辆说明

  7. C0

    传统语音控制

    规则命令、关键词识别，无大模型证据。

    用户感知车能听命令

    示例打开空调、调座椅

使用规则

  • Current Level只按已确认或强证据能力评分。
  • Potential Level可用于厂商宣称、后续 OTA 或概念展示。
  • 证据不足时标注 Unknown，不强行评分。
  • 同一品牌可出现「当前 C3，潜在 C5」的判断。
  • 总矩阵中的 Cx–Cy 表示能力跨档或证据不完全收敛：最低保证 Cx，最高观测到 Cy。
  • 智驾模型、舱驾综合供应商和本届弱发布对象进入观察表，不使用 C0–C6 座舱阶梯打分。

§7.2核心能力清单 · 11 项

每项能力都标注「人无我有 / 人有我优」并给出对外价值与竞品对比意义。前 8 项为差异化主能力，后 3 项为体系完整度支撑。

1.1

生态闭环 Agent / Agent 闭环

重点投入 · 高可控 · 可编排

AI 通过生态打通，像人一样跑完整段第三方 App 任务——AI 长出手。

  • 高可控每一步可见、可干预、可回退，区别于黑盒视觉自动化
  • 可编排Planner 把长指令拆成 Agent 可执行的子任务，像剧本一样串成闭环
  • 类人操作替用户搜店、订座、买券、下单、叫跑腿，跨 App 完成生活任务
  • 泛化能力学会点外卖后，自动迁移到订餐 / 买券 / 叫跑腿等同结构任务
  • 自主决策碰到广告自动关闭，误操作自动换路径

对外价值AI 真的长出手

路径与口径走协议级生态共建而非视觉抓屏路线——技术上需要与具备视觉感知 / UI 操作能力的第三方生态共建，跨 App 闭环的节奏取决于合作进度。对外用「生态闭环 Agent / AI 长出手」这一组叙事，避免直接挪用「读屏自动化 ／ 行业首发 ／ 人无我有」等暂不可核验的绝对化口径。

竞品对比所有竞品仍停留在「对话式问答」，无可持续操作工作区——跟进需 12–18 个月。

1.5

活人感

人有我优

声音 + 情绪 + 人格化。

  • 语音合成 2.0：「人机难辨」级中文对话
  • 喜怒哀乐多情感参数、悄悄话模式、气音、儿化音
  • 豆包机器人（桃子形象）+ DMS 摄像头主动识别
  • 全时免唤醒 + 多人交互

对外价值她不像工具

竞品对比对比 NOMI、理想同学、讯飞——他们已经是市场强参照，自有方案要靠「物理机器人 + DMS + 多人识别」做组合差异。

1.6

音乐与方言

人有我优

即兴创作、翻唱、方言识别 / 输出。

  • 灵魂歌手模式：即兴创作 + 11 种音乐风格 + 翻唱
  • 合唱对唱：音乐翻唱，情绪同步
  • 4 种方言输出（粤语 / 四川话 / 东北话 / 陕西话）+ 18 种方言识别

对外价值娱乐和地域共鸣

竞品对比部分玩家具备方言、识别——自有方案的差异在「歌唱 + 合唱 + 多方言输出」的组合传播力。

1.7

AI 氛围灯

人无我有

像素级灯珠控制 + 识图控灯 + 意图理解编排 + 音乐联动。

  • 像素级灯珠控制（竞品只能改整车颜色）
  • 识图控灯：AI 理解外部环境映射到车内灯光
  • 「把孩子的画变成车内光影」
  • 意图理解编排：「来点浪漫的氛围」→ 自动编排灯光方案
  • 音乐联动：灯光随节奏和情绪变化

对外价值情绪物理化

竞品对比对比普通氛围灯与硬件联动——竞品多是整车颜色变换，自有方案是像素级 + 语义编排。

1.8

AI Push · 前置提醒

人有我优

把「下一步」前置到眼前。

  • 到达公司前主动推送会议提醒
  • 结合导航 ETA 自动调整日程提醒时机

对外价值主动服务，不是被动响应

竞品对比对比日程 / 导航提醒——多数车机仍是被动响应或固定时间提醒，自有方案是「上下文感知的主动 Push」。

1.9

长期记忆

人有我优

该记才记，不该记坚决不记。

  • 记得用药时间、喜欢的音乐、常去的地方
  • 家庭成员关系、特殊需求
  • 与 AI Push 联动，在合适场景主动调用

对外价值真正懂你

竞品对比NOMI、理想等有基础——自有方案差异在「记忆边界 + 隐私守门」的可信度，避免一味堆功能。

1.10

深度思考模型

体系基底

复杂推理与端云协同。

  • 豆包 2.0 Pro 可作为深度思考能力的传播锚点，具体对标对象需按发布时公开 benchmark 口径展示
  • SuperGPQA、HealthBench 等榜单结论暂列为待核验素材，不直接写成无来源强事实
  • Token 定价和成本优势需保留来源、测试条件和时间口径
  • OTA 持续升级

对外价值能处理复杂需求

证据口径模型 benchmark 只作为辅助证明，发布前必须绑定官方榜单、测试日期和原始链接；主表达仍落到"任务表现"的真实演示。

1.11

完整架构 · 系统集成

体系基底

Planner + Agents + 生态闭环 Agent + 地图/车控/记忆深度融合，端云协同 + 隐私 + OTA。

  • 用户意图输入（语音/文字/视觉）→ Planner → 生态闭环 Agent / AI 地图 Agent / 车控 Agent → 多模态融合层 → 深度思考 + 记忆 + 隐私 → 车辆执行层
  • 与 Tesla + Grok 的核心差异：Grok 是独立助手与车控分离，豆包是深度融合

对外价值从 Chatbot 到 Agent 的真正跃迁

竞品对比系统级整合参照华为；架构语言要拆出 Planner + Agents + 生态闭环 Agent，避免被竞品「也讲 Agent」淹没。

§7.3三层情绪与能力结构 · A → B → C

B 组

她真懂你

认可 → 触动

从「理解你说的」跳到「理解你没说的」。理性需求背后的感性需求，她都接得住。

C 组

她在车里活了

触动 → 震撼

证明 AI 不只是屏幕里的助手，而是车内物理空间中的角色。她在物理空间里有位置、有朝向、有注视。

§10.4最终建议 · Final Take

自有内容应避免陷入「谁的大模型参数更强」的横向比较。行业已经进入大模型普及期，单纯说「接入大模型」很快会失去差异。最值得坚持的主线是——

从 Chatbot 到 Agent，
从「能回答」到「能执行」，
从车机助手到车内角色。

具体到产品和传播，应以四个锚点支撑：

  1. 01

    Planner

    能理解复杂需求并拆解任务。

  2. 02

    生态闭环 Agent

    高可控、可编排，借助生态打通让 AI 长出手，跨 App 跑完一整段任务闭环。

  3. 03

    AI 地图 Agent

    能把地图、POI、内容和路线变成体验规划。

  4. 04

    活人感

    能通过声音、记忆、情绪、机器人和硬件联动成为车内角色。

最终记忆点保持简单：

豆包上车，AI 替你动手。
